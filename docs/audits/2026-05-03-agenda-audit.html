7 · 🔥 Gaps + Recomendações

Gap #1 · Secretaria não tem UI pra criar/editar appointment direto

Hoje a secretaria recebe pedido de horário no chat, vai pra /crm/agenda (interface técnica), agenda manualmente lá. Cliques: muitos. Persona idosa: travada.

Solução: Sprint 2 do roadmap · botão "Criar agendamento" direto na /secretaria · wizard mini de 3 passos:

  1. Procedimento (dropdown com top 8 da clínica)
  2. Data + horário (calendário com slots livres do profissional)
  3. Confirmar → cria appointment + dispara mensagem confirmação

Reusa lead_to_appointment RPC existente · zero schema novo.

Gap #2 · 0 appointments futuros no DB (só 3 finalizados)

Clínica está usando agenda externa (papel/Google Calendar) e não trouxe pra cá. Resultado: lembretes automáticos não disparam (sem appointments) · automations on_status nunca rodam · UI /crm/agenda vazia.

Solução: migração assistida da agenda atual (planilha/papel) pra o sistema. Ou política "todo agendamento novo entra no v2 a partir de X". Sem isso, máquina de automation não roda.

Gap #3 · wa_agenda_automations sem UI de gerenciamento na Lara v2

68 automations ativas vivem no banco · admin não consegue editar pelo Lara v2. Quando precisar mudar texto de lembrete D-1, dev tem que mexer SQL.

Solução: rota /configuracoes/automacoes (admin only) com lista + editor de templates + toggle ativa. RPCs já existem (wa_agenda_auto_list/upsert/delete/toggle).

Gap #4 · agenda_alerts_log vazio apesar do cron rodar

agenda_alert_min_before_tick roda cada minuto MAS log fica vazio · provável: 0 appointments futuros pra disparar. Confirmação que máquina espera dados.

Solução: validar com 1 appointment teste · ver se log popula. Se sim, está OK · só falta volume.

Gap #5 · Sem appointment_history table

chegada_em, cancelado_em, no_show_em trackeiam eventos finais · mas transições intermediárias (agendado→aguardando_confirmacao→confirmado→...) não tem audit trail. Útil pra métricas tipo "tempo médio confirmação" ou "taxa cancelamento por status".

Solução proposta (V2): tabela appointment_status_history com (appointment_id, from_status, to_status, changed_at, changed_by, reason). Trigger AFTER UPDATE OF status registra automático.

Gap #6 · Indexes compostos faltando

Hoje tem 8 indexes · suficientes pro caso single-clinic (Mirian). Mas pra multi-tenant futuro:

  • (clinic_id, scheduled_date, status) · dashboards "agendados pra hoje"
  • (clinic_id, status) WHERE status IN ('aguardando_confirmacao','confirmado') partial · lembretes

8 · O que está bom · não mexer

  • Schema · 35 colunas · zero excesso · todo campo com propósito
  • State machine validada por RPC _appointment_status_transition_allowed
  • Triggers de phase · trg_appointment_attended_phase + trg_appointment_created_phase + trg_appointment_rescheduled_phase · automaticamente bumpa lead.phase
  • VPI integration · _vpi_ind_stage_on_appointment · attribution rastreada
  • Cashflow integration · cashflow_link_appointment · receita ligada ao agendamento
  • RPCs WA · wa_pro_stage_create/cancel/reschedule_appointment · permite Mira/Lara criarem appointment via chat
